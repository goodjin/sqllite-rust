## 1. 系统定位

### 1.1 系统概述

sqllite-rust 是一个使用 Rust 语言实现的嵌入式关系型数据库，功能对标 SQLite 核心引擎。采用单文件存储设计，支持标准 SQL 查询、事务、索引等核心功能。

**核心特性**:
- 单文件数据库存储（类似 SQLite 的 `.db` 文件）
- B+ Tree 存储引擎
- SQL-92 子集支持
- ACID 事务（通过 WAL 实现）
- 纯 Rust 实现，内存安全

### 1.3 模块划分

| 模块编号 | 模块名称 | 职责 | 对应PRD |
|---------|---------|------|---------|

| MOD-02 | Pager | 页面管理，缓存，I/O 抽象 | FR-002 |

| MOD-05 | Transaction Manager | ACID 事务，WAL 日志 | FR-014~015, FR-017 |
| MOD-06 | Index Manager | B+ Tree 索引管理 | FR-016 |

## 2. 技术选型

| 层次 | 技术栈 | 选型理由 |
|-----|-------|---------|
| 编程语言 | Rust 1.70+ | 内存安全、零成本抽象、优秀的错误处理 |
| 存储格式 | 自定义二进制格式 | 参考 SQLite，单文件设计 |
| 页面大小 | 4096 字节 | 与操作系统页对齐，I/O 效率最优 |
| 并发模型 | 读写锁 (RwLock) | 多读者单写者，简单高效 |
| 错误处理 | thiserror + anyhow | 结构化错误，便于调试 |
| 测试框架 | 内置 test + criterion | 单元测试和基准测试 |

## 3. 三阶段里程碑架构

### 阶段 1: 存储引擎 (MVP)

**目标**: 实现基础存储和 CRUD

```
┌─────────────────┐
│   Storage API   │  ← 直接 API 调用
└────────┬────────┘
         │
┌────────▼────────┐
│    B+ Tree      │
│   (MOD-01)      │
└────────┬────────┘
         │
┌────────▼────────┐
│     Pager       │
│   (MOD-02)      │
└────────┬────────┘
         │
┌────────▼────────┐
│    File I/O     │
└─────────────────┘
```

**交付物**:
- B+ Tree 增删改查
- 页面缓存管理
- 基础数据类型 (INTEGER, TEXT)

### 阶段 2: SQL 层

**目标**: 支持 SQL 查询

```
┌─────────────────┐
│   SQL String    │
└────────┬────────┘
         │
┌────────▼────────┐
│  SQL Parser     │
│   (MOD-03)      │
└────────┬────────┘
         │ AST
┌────────▼────────┐
│    VM / Code    │
│   Generator     │
└────────┬────────┘
         │ Bytecode
┌────────▼────────┐
│  Virtual Machine│
│   (MOD-04)      │
└────────┬────────┘
         │
    [Storage Layer]
```

**交付物**:
- SQL 解析器
- 字节码虚拟机
- SELECT/INSERT/UPDATE/DELETE
- WHERE 子句

### 阶段 3: 高级功能

**目标**: 生产就绪

```
┌─────────────────────────────────────────┐
│           SQL Interface                 │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────▼──────────────────────┐
│  Parser → Optimizer → VM → Storage      │
└──────────────────┬──────────────────────┘
                   │
    ┌──────────────┼──────────────┐
    ▼              ▼              ▼
┌────────┐   ┌──────────┐   ┌──────────┐
│  WAL   │   │Transaction│   │  Index   │
│  File  │   │  Manager  │   │  Manager │
└────────┘   └──────────┘   └──────────┘
```

**交付物**:
- 事务支持 (ACID)
- WAL 预写日志
- B+ Tree 索引
- 并发控制

## 6. 设计原则

### 6.1 内存安全
- 使用 Rust 的所有权系统避免内存泄漏
- 避免 unsafe 代码，除非必要且经过严格审查
- 使用 RAII 模式管理资源

### 6.2 模块化设计
- 各模块职责单一，接口清晰
- 模块间通过明确定义的 API 交互
- 支持独立测试各模块

### 6.3 错误处理
- 使用 Result 类型传递错误
- 错误信息包含上下文，便于调试
- 区分可恢复错误和不可恢复错误

### 6.4 性能考虑
- 页面缓存减少 I/O
- B+ Tree 优化范围查询
- WAL 提高并发写入性能

## 7. 风险评估

| 风险 | 影响 | 缓解措施 |
|-----|------|---------|
| B+ Tree 实现复杂度高 | 高 | 参考 SQLite 和 sled 的实现，充分测试 |
| WAL 恢复逻辑复杂 | 中 | 详细设计恢复流程，编写单元测试 |
| 性能不达预期 | 中 | 早期进行基准测试，持续优化 |
| 兼容性问题 | 低 | 严格遵循 SQLite 文件格式规范 |
