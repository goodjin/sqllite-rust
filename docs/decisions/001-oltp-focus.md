# 决策记录: OLTP 聚焦

## 背景

最初的项目规划包含了向量化执行、列式存储、GPU 加速等优化方向。

## 问题

- **向量化**、**列式存储**、**GPU 加速** 是 OLAP（分析型）技术
- **SQLite 是 OLTP（事务型）**数据库
- 方向不匹配

## 决策

**放弃向量化/列式/GPU，专注于 OLTP 核心优化。**

### 新的优化方向

| 优先级 | 优化项 | 预期提升 |
|--------|--------|---------|
| P0 | 预编译缓存 | 3-5x |
| P0 | MVCC 无锁读 | **100x** |
| P1 | B+Tree 缓存 | 2-3x |
| P1 | WAL 组提交 | 2-5x |
| P2 | 查询优化器 | 2-10x |
